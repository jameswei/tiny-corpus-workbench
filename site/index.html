Purpose and workflow

Keep the extraction evidence visible.

Content and structure can be lost before downstream use. The workbench makes that handoff inspectable without changing the source.

  1. 01 Original source Unchanged
  2. 02 Validated private snapshot Temporary; removed before publication
  3. 03 Docling / MarkItDown Docling JSON is canonical; Markdown views are derived
  4. 04 Manifest + comparison Recorded checks and extractor outputs
  5. 05 Published evidence Inspectable and read-only to the application

Extraction Observatory

A focused v0.1 surface.

Observe PDF, DOCX, Markdown, and plain-text fixtures locally after PDF model prefetch. Twelve deterministic fixtures make comparisons repeatable.

Two explicit commands

  • tcw observe creates an observation.
  • tcw verify checks published evidence.
  • Observation disables remote services, plugins, OCR, and LLM clients.

Integrity boundary

Know what verification means.

The verifier checks evidence within a trusted local boundary. Its claim is deliberately narrow.

Verified

Evidence consistency

Structural validity, artifact integrity, and explicitly derivable semantic consistency.

Not authenticated

Creator identity

The verifier does not establish who created or last changed otherwise valid evidence.

Quick start and learning

Run one local observation.

Use CPython 3.12 and the exact dependency lock. Prefetch the required PDF models before the offline observation.

uv sync --frozen --python 3.12
uv run --frozen docling-tools models download layout tableformer --output-dir .cache/docling/models
uv run --frozen tcw observe fixtures/golden/policy-memo.pdf
